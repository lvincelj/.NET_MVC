# Semantic Model

## EF Entities and Tables

### Patient (table: Patients)
- Main properties: Id, FirstName, LastName, Gender, DateOfBirth, Email, PhoneNumber, Address
- Foreign keys: none
- Navigation properties: Appointments, MedicalRecords

### Doctor (table: Doctors)
- Main properties: Id, FirstName, LastName, Gender, Specialty, Email, PhoneNumber
- Foreign keys: none
- Navigation properties: Departments, Appointments

### Department (table: Departments)
- Main properties: Id, Name, Location, PhoneNumber, HeadOfDepartment
- Foreign keys: none
- Navigation properties: Doctors

### Appointment (table: Appointments)
- Main properties: Id, PatientId, DoctorId, ScheduledAt, Status, Room, Notes
- Foreign keys: PatientId -> Patients.Id, DoctorId -> Doctors.Id
- Navigation properties: Patient, Doctor

### MedicalRecord (table: MedicalRecords)
- Main properties: Id, PatientId, CreatedAt, Diagnosis, Notes
- Foreign keys: PatientId -> Patients.Id
- Navigation properties: Patient, Prescriptions

### Prescription (table: Prescriptions)
- Main properties: Id, MedicalRecordId, IssuedAt, IssuedBy
- Foreign keys: MedicalRecordId -> MedicalRecords.Id
- Navigation properties: MedicalRecord, Medications

### Medication (table: Medications)
- Main properties: Id, Name, Dosage, Instructions, PrescriptionId
- Foreign keys: PrescriptionId -> Prescriptions.Id
- Navigation properties: Prescription

### DoctorDepartment (join table: DoctorDepartments)
- Purpose: many-to-many bridge between Doctors and Departments
- Keys/columns: DoctorId, DepartmentsId (generated by EF for the implicit join entity)
- Configured in AppDbContext.OnModelCreating

## Supporting Models and Enums (not mapped as core tables)

### AppointmentStatus (enum)
- Values: Scheduled, CheckedIn, InProgress, Completed, Cancelled, NoShow
- Used by: Appointment.Status

### Gender (enum)
- Values: Unknown (0), Male (1), Female (2)
- Used by: Patient.Gender, Doctor.Gender

### ErrorViewModel
- Properties: RequestId, ShowRequestId
- Role: MVC error page view model

### HomeDashboardViewModel
- Properties: TotalPatients, ActiveDoctors, DepartmentCount, TodayAppointments, PendingRecords, MedicationCount, Actions, UpcomingAppointments, OperationalNotes
- Role: dashboard page projection model

### DashboardActionItem
- Properties: Title, Description, Controller, Accent, Metric
- Role: dashboard UI action card model

### DashboardAppointmentItem
- Properties: TimeLabel, PatientName, DoctorName, Room, Status
- Role: dashboard UI appointment summary model

### DashboardNoteItem
- Properties: Title, Detail, Tone
- Role: dashboard UI operational note model

## Relationship Summary
- Patient 1 -> many Appointment
- Doctor 1 -> many Appointment
- Patient 1 -> many MedicalRecord
- MedicalRecord 1 -> many Prescription
- Prescription 1 -> many Medication
- Doctor many <-> many Department (via DoctorDepartments)
